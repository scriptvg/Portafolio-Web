# Debugging de Responsividad en Producción

## Herramientas de Testing

### Chrome DevTools
1. Abrir DevTools (F12)
2. Toggle Device Toolbar (Ctrl+Shift+M)
3. Probar diferentes dispositivos:
   - iPhone SE (375px)
   - iPhone 12 Pro (390px)
   - Pixel 5 (393px)
   - Samsung Galaxy S20 (412px)
   - iPad Air (820px)
   - iPad Pro (1024px)

### Firefox Responsive Design Mode
1. Abrir DevTools (F12)
2. Toggle Responsive Design Mode (Ctrl+Shift+M)
3. Probar orientaciones portrait/landscape

### Safari (para iOS)
1. Usar Safari en Mac con simulador iOS
2. Probar en dispositivos reales iOS

## Checklist de Problemas Comunes

### ❌ Overflow Horizontal
**Síntomas**: Scroll horizontal no deseado
**Solución**: 
```css
body, #root {
  overflow-x: hidden;
  max-width: 100vw;
}
```
✅ Ya implementado en `src/index.css`

### ❌ Elementos Fixed que se Salen
**Síntomas**: Navbar o elementos fixed más anchos que la pantalla
**Solución**:
```jsx
className="fixed inset-x-4 max-w-7xl mx-auto"
```
✅ Ya implementado en `ProfessionalNavbar.jsx`

### ❌ Imágenes que Rompen el Layout
**Síntomas**: Imágenes más grandes que su contenedor
**Solución**:
```css
img {
  max-width: 100%;
  height: auto;
}
```
✅ Ya implementado en `src/index.css`

### ❌ Texto Muy Pequeño en Móviles
**Síntomas**: Texto ilegible en dispositivos móviles
**Solución**:
```jsx
className="text-xs sm:text-sm md:text-base lg:text-lg"
```
✅ Ya implementado en todos los componentes

### ❌ Botones Muy Pequeños para Touch
**Síntomas**: Difícil hacer click en móviles
**Solución**:
```jsx
// Mínimo 44x44px para touch targets
className="h-10 w-10 sm:h-12 sm:w-12"
```
✅ Ya implementado en componentes interactivos

### ❌ Grid que No Se Adapta
**Síntomas**: Columnas muy estrechas en móvil
**Solución**:
```jsx
className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3"
```
✅ Ya implementado en todos los grids

## Problemas Específicos de Producción

### 1. CSS No Se Aplica Correctamente
**Causa**: Orden de importación o especificidad
**Solución**: Verificar orden en `main.jsx`:
```jsx
import './index.css'  // Primero
import './App.css'    // Después
```

### 2. Animaciones Lentas en Móviles
**Causa**: Framer Motion con muchas animaciones
**Solución**: Ya implementado `prefers-reduced-motion` en `App.css`

### 3. Fuentes No Cargan
**Causa**: CORS o ruta incorrecta
**Solución**: Verificar en Network tab de DevTools

### 4. Imágenes No Cargan
**Causa**: Rutas relativas incorrectas
**Solución**: Usar rutas absolutas desde `/src/assets/`

## Solución Rápida de Emergencia

Si después de desplegar encuentras problemas de responsividad:

1. **Verificar viewport meta tag**
   ```html
   <meta name="viewport" content="width=device-width, initial-scale=1.0">
   ```

2. **Agregar overflow-x hidden**
   ```css
   body { overflow-x: hidden; }
   ```

3. **Verificar max-width en contenedores**
   ```css
   .container { max-width: 100vw; }
   ```

4. **Rebuild y redeploy**
   ```bash
   pnpm build:clean
   # Deploy según tu plataforma
   ```
